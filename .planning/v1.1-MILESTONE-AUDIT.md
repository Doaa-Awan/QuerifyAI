### Integration Findings (Non-Critical)

**INT-01: requireSession not wired to production routes**
- `requireSession` is implemented in `server/middleware/requireSession.js`, correctly tested (TEST-04), but never imported in `server/routes.js`.
- Routes `/db/schema`, `/api/chat`, `/db/explorer-context/snapshot`, and others are reachable without a session check.
- **This is not a v1.1 gap.** AUTH-03 (apply requireSession to guarded routes) is explicitly listed as a deferred requirement in REQUIREMENTS.md. The v1.1 scope for AUTH-01/AUTH-02 was only that the flag is reliably SET — not enforced. The enforcement wire is the next milestone's work.
- **INT-02 (minor):** `connectAndIntrospect` Zod 400 path returns without setting `req.session.connected = false`. If a prior session existed and the user sends a malformed request, the old flag is not cleared. Low severity since Zod failures don't change DB state.

**FINDING-03 (doc gap):** Route path documentation mismatch — milestone docs reference `/db/connect-and-introspect` but `routes.js` registers the third postgres connect route as `POST /api/connect`. Similarly, MSSQL routes are `/db/connect-sqlserver` and `/db/connect-demo-sqlserver`, not `/db/connect/mssql` and `/db/connect-demo/mssql`. Functional wiring is correct; documentation needs updating before v1.2.
