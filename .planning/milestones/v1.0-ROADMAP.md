# Milestone v1.0: Portfolio MVP

**Status:** ✅ SHIPPED 2026-03-26
**Phases:** 1-2

## Overview

Transform the working Querify prototype into a deployed, demo-ready portfolio piece: clean query API, memory safety caps, full deployment stack on Vercel + Railway, and demo-facing UX (cold start banner, rate limit banner, ERD visualizer, SQL Server support).

**Requirements**: QAPI-01, QAPI-02, QAPI-03, QAPI-04, MEM-01, MEM-02, MEM-03
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md — POST /api/query endpoint, cache.js (200-entry FIFO SHA-256), query controller, deprecation warn on /api/chat
- [x] 01-02-PLAN.md — Cap topicCache (100 entries), conversations Map (200 entries / 20 msgs/conv), switch ChatBot.jsx to /api/query

## Milestone Summary

**Key Decisions:**
- `reactflow@11` used instead of `@xyflow/react v12` — peer dep uncertainty with React 19; same underlying library
- Custom layered layout instead of `@dagrejs/dagre` — sufficient for demo scale without extra dependency
- SQL Server support added via quick tasks rather than formal phase — low-risk, mirrors PostgreSQL pattern
- `sameSite:'none'` in production via NODE_ENV guard — required for Vercel+Railway cross-origin sessions
- SHA-256 cache key on table names — column-level changes don't bust cache (documented tradeoff)

**Issues Resolved:**
- Chat follow-up detection had false positives — removed trigger phrases ('also'), improved synonym matching
- CI/CD was deploying before tests passed — fixed GitHub Actions pipeline ordering
- Rate limit headers were using new standard format — switched to legacy `X-RateLimit-*` for frontend compatibility

**Issues Deferred:**
- DEPL-06: Supabase demo DB env vars not set in Railway
- RateLimitBanner missing time-until-reset display in blocked state
- `apiFetch` helper in api.js is dead code

**Technical Debt Incurred:**
- No VERIFICATION.md files created for either phase (executed before GSD Nyquist validation workflow)
- ERD spec deviations: reactflow@11, custom layout, no MiniMap, no row count badge
