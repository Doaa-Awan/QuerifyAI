# Plan 03-01: Set Session Flag in All Connect Handlers

## Goal

Add `req.session.connected = true/false` to every connect handler so that the session flag is reliably set after a successful connection and cleared after a failed one.

## Context

The flag was only wired into `connectAndIntrospect` (the composite connect+introspect endpoint). The simpler `connectDemo` and `connect` handlers in both the Postgres and MSSQL controllers never set the flag. Additionally `connectAndIntrospect` never cleared the flag on failure.

## Tasks

<task id="1">
Modify `postgresController.connectDemo` in `server/controllers/postgres.controller.js`:
- On success (`result.ok`): add `req.session.connected = true;` before `res.json()`
- On failure: add `req.session.connected = false;` before `res.status(...).json()`
</task>

<task id="2">
Modify `postgresController.connect` in `server/controllers/postgres.controller.js`:
- On success (`result.ok`): add `req.session.connected = true;` before `res.json()`
- On failure: add `req.session.connected = false;` before `res.status(...).json()`
</task>

<task id="3">
Modify `postgresController.connectAndIntrospect` in `server/controllers/postgres.controller.js`:
- On failure: add `req.session.connected = false;` before `res.status(...).json()` (success already sets `true`)
</task>

<task id="4">
Modify `mssqlController.connectDemo` in `server/controllers/mssql.controller.js`:
- On success: add `req.session.connected = true;` before `res.json()`
- On failure: add `req.session.connected = false;` before `res.status(...).json()`
</task>

<task id="5">
Modify `mssqlController.connect` in `server/controllers/mssql.controller.js`:
- On success: add `req.session.connected = true;` before `res.json()`
- On failure: add `req.session.connected = false;` before `res.status(...).json()`
</task>

## Verification Criteria

- `POST /db/connect-demo` succeeds → `req.session.connected === true`
- `POST /db/connect` with unreachable host → `req.session.connected === false`
- `POST /db/connect-demo-sqlserver` succeeds → `req.session.connected === true`
- `POST /db/connect-sqlserver` with bad config → `req.session.connected === false`
- `POST /api/connect` fails → `req.session.connected === false`

## must_haves

- Every successful connect handler sets `req.session.connected = true`
- Every failed connect handler sets `req.session.connected = false`
- No existing handler logic is changed beyond adding the flag lines
