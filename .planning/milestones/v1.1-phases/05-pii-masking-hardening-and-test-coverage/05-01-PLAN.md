<objective>
Fix four PII masking gaps in server/services/postgres.service.js:
1. isLikelyPiiColumn incorrectly skips date-typed columns before checking PII name patterns, causing `dob`/`birth_date` date columns to be sent raw to the LLM
2. buildDummyValue has no SSN branch — integer SSN values fall through to `return n` (a real number)
3. buildDummyValue has no passport branch — passport values fall through to a generic string redaction
4. generateTableDescriptions has an uncaught JSON.parse that propagates exceptions on malformed AI responses

Purpose: Close the PII gaps so the LLM never receives real SSN, DOB, or passport values, and snapshot generation survives AI parse failures.
Output: Modified server/services/postgres.service.js with all four gaps closed.
</objective>
