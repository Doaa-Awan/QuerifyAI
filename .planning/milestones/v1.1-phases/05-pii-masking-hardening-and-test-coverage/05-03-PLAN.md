    IMPORTANT — rate limiter store approach:
    express-rate-limit v8 default MemoryStore is keyed on req.ip.
    To test exhaustion, call the middleware 11 times with the same IP.
    First 10 calls should invoke next(). The 11th should invoke the handler (429).

    To isolate test state, create a fresh connectLimiter instance inside the
    test (import rateLimit from express-rate-limit and rateLimitHandler logic),
    OR import the connectLimiter from rateLimiter.js and reset its store between
    tests using connectLimiter.resetKey('127.0.0.1') if available.

    SIMPLEST approach: import connectLimiter from rateLimiter.js directly and
    call it with mock req/res. express-rate-limit v8 middleware signature:
      await connectLimiter(req, res, next)  — it is an async middleware.

    Mock req must include:
    - ip: '127.0.0.1'
    - headers: {} (rate limiter reads headers)
    - socket: { remoteAddress: '127.0.0.1' }

    - url: '/db/connect'
    - app: { get: () => false }
