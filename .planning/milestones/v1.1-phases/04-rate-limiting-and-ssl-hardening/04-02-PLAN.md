<objective>
Make PostgreSQL SSL certificate validation configurable via `POSTGRES_SSL_REJECT_UNAUTHORIZED` environment variable. Secure by default (rejectUnauthorized: true when the var is absent or 'true'); opt-out for dev/self-signed cert environments by setting the var to 'false'.

Purpose: The current hardcode `rejectUnauthorized: false` silently downgrades all SSL connections to unverified, which is insecure in production. The fix makes production-safe the default while preserving opt-out for development.
Output: Updated createPostgresClient() function, updated server/.env.example with new var documented.
</objective>

In server/services/postgres.service.js, modify the `createPostgresClient` function (lines 26-36).

Replace the hardcoded `ssl` property with an env-var-driven one:

```js
function createPostgresClient(config) {
  const rejectUnauthorized = process.env.POSTGRES_SSL_REJECT_UNAUTHORIZED !== 'false';
  return new Pool({
    host: config.host,
    port: config.port,
    user: config.user,
    password: config.password,
    database: config.database,
    ssl: config.ssl ? { rejectUnauthorized } : false,
    ...(config.options && { options: config.options }),
  });
}
```

Logic:
- `process.env.POSTGRES_SSL_REJECT_UNAUTHORIZED !== 'false'` evaluates to `true` when the var is absent, empty, or any value other than the exact string `'false'`
- Only the exact string `'false'` opts out — prevents accidental opt-out from typos
- The `config.ssl` outer condition is unchanged — SSL is still only used when the caller passes `ssl: true`

Do NOT change any other part of the function or the rest of the file.
