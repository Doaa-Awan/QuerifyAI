<objective>
Harden the server and client for cross-origin deployment on Railway + Vercel.

Purpose: Railway (backend) and Vercel (frontend) are different origins. Without sameSite=none, secure=true, and trust proxy, all session cookies are silently dropped after deploy. The GET /health route is required for Railway's deploy health check and the cold start poller in Plan 02. api.js centralises API_BASE so one env var change wires all components. vercel.json and railway.toml encode deploy config as files rather than dashboard clicks.

Output: Session cookie hardening, GET /health route, api.js module, rateLimiter emitting X-RateLimit-Remaining, vercel.json and railway.toml deploy configs.
</objective>
